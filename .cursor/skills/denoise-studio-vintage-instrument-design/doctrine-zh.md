# 复古电子仪器美学 — 中文教义

## 第一章 为什么米白 + 绿屏有效

人对「测量仪器」的深层图式来自物理世界：实验室台式设备多为 **浅色塑胶前面板**（反光低、耐脏、印刷丝印清晰），显示区域为 **深色荧光屏**（自发光、对比极高）。米白与磷光绿的组合因此自带「可读性 + 权威感」，比通用 Web 灰白卡更接近「这台机器在测信号」。

网页 UI 失败，往往是因为 **机身与屏幕对比不足**（灰底灰字），或 **绿色过饱和像 Matrix 屏保**。本 skill 要求：机身暖、屏幕暗、绿色有辉光但不刺眼。

## 第二章 注塑机身的心理学

米白 `#e8e4da` 不是纯白——纯白像医院或苹果店；略带黄灰像 ABS 老化、像学校实验室示波器。机身边缘略深、顶部略亮，模拟注塑脱模后的微弱高光。用户不必知道注塑工艺，但 **潜意识会读作硬件**。

避免在整个 RackPanel 外卡涂米白——那会变成「黄色大块塑料网站」。复古只涂在 **内嵌仪器模块**（示波器区、transport 条），外框仍用 Signal Console 玻璃机架，形成「机架里的老仪器」叙事，这与 Denoise Studio 论文演示场景也吻合：现代 web 平台里嵌一块经典测量 UI。

## 第三章 磷光屏的物理隐喻

CRT 磷光体发光后有余辉（persistence）。UI 上用：

- 波形略粗 + 外晕模拟余辉
- 扫描线暗示电子束横向扫过
- 网格暗、波形亮，形成「信号跳出网格」的层次

idle 无信号时，仍应有 **暗绿底 + 暗网格 + 丝印提示**，像开机但未进线的示波器，而非「组件加载失败」。

## 第四章 丝印字与 engraved 效果

真仪器型号、通道、单位是丝网印刷或激光雕刻，特征：小号、全大写、字距宽、颜色比机身深一度。CSS 用 mono 字体 + `text-shadow` 双向微阴影做出略凹进面板的感觉。`CH1 / TIME`、`STBY`、`dBFS` 比段落说明更适合丝印风格。

中文标签（如「实时预览」）仍用 i18n 正文体；**通道与状态缩写**用英文丝印更符合复古仪器惯例，中英文答辩可并存。

## 第五章 Pilot 灯与 REC

红色 REC 灯是跨文化录音符号。复古化处理：

- 灯体在凹座里（inset）
- 点亮时外晕（box-shadow spread 小）
- 灯灭时暗红半透明，非灰色圆点

这与 Flat UI 的红色按钮不同——它是 **状态指示器**，不是主 CTA。主操作仍可点 REC 文字区，但视觉中心是 pilot。

## 第六章 扫描线、辉光与无障碍

扫描线宜慢（周期 ≥ 1s 横穿），alpha ≤ 0.15。`prefers-reduced-motion: reduce` 时关闭扫描线与呼吸辉光，保留静态磷光波形与网格，避免光敏不适。

## 第七章 与现有 Live 页整合

用户已删除 MOD-07B 耳机监听模块，保留 MOD-07A 示波器——**最适合全套复古化的单焦点**。建议：

- `live-module-input` + `vintage-instrument`
- 示波器高度充足（≥ 160px）
- VU 在机身底部像前面板 meter
- ANC 一行控制做成小型 toggle 或 dual pilot，勿再加大拟物

L1/L2/L3 tier 与 flow 保持 Signal Console，不改成老式拨杆，避免信息架构倒退。

## 第八章 代码纪律

- 新样式进 `vintage-instrument.css`
- Canvas 读 CSS 变量，不硬编码 `#3dff7a`
- 不在 PNG 贴 CRT 边框——用 CSS bezel + inset
- 不引入 Three.js 做 CRT 曲面

## 第九章 答辩与论文

插图若展示 Live 页，caption 可写「复古示波器式实时波形监视（磷光绿轨迹）」；与「轻量 OM-LSA 实时预览」技术描述并列，体现 **HMI 设计意图**。勿在论文中写「拟物耳机」若 UI 已删除。

## 第十章 常见用户语句 → 改法

复古电子仪器 front-end 不是「旧照片滤镜」，而是 **材料 + 发光显示 + 丝印逻辑** 三件套。掌握本 skill，Agent 可在不破坏 Signal Console 机架的前提下，把 Denoise Studio 的示波器区做成评委一眼认出的 **米白机身、绿色荧光屏** 实验室设备。
